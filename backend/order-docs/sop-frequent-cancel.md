# SOP：频繁下单/取消异常处理

## 适用场景
- 同一用户在短时间内频繁下单并取消，影响库存/运营指标，或疑似刷单/探价。

## 判定标准（常用）
- 24 小时内下单次数异常（例如 ≥ 5）
- 取消率异常（例如 ≥ 50%）
- 多次更换收货地址/收件人/联系方式

## 处理步骤（最小流程）
1. 标记用户风险（进入观察名单）。
2. 检查取消原因（是否支付失败、是否频繁改价/优惠叠加失败）。
3. 对高频用户：限制发放优惠（如有），或降低营销触达频率（按运营策略）。
4. 如疑似恶意：对后续订单进入人工审核（必要时限制下单能力，需产品/运营确认规则）。
5. 记录结论：异常原因与处理动作。

## 注意事项
- 先区分“系统原因/支付问题”与“恶意行为”，避免误伤正常用户。
- 涉及限制下单等强动作，需明确规则与留痕。
