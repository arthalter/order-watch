# SOP：同地址多账号下单异常处理

## 适用场景
- 同一收货地址在短时间内出现多个账号下单（尤其是新账号），疑似羊毛/团伙。

## 判定标准（常用）
- 24 小时内同地址关联账号数异常（例如 ≥ 3）
- 账号新注册、无历史订单
- 订单集中在高补贴/高折扣商品
- 联系方式/收件人信息相似（同名、同手机号段等）

## 处理步骤（最小流程）
1. 归并同地址订单（按地址维度拉清单）。
2. 核对优惠使用情况（是否异常集中、是否重复薅券）。
3. 对高风险订单：暂缓发货 → 人工审核（必要时电话核验）。
4. 对确认异常的订单：取消/拦截并记录原因；视情况拉黑地址/账号（按策略）。
5. 输出“关联关系说明”作为留痕：账号列表、订单号、关键证据点。

## 注意事项
- 注意“正常合住/公司地址/宿舍”等合理场景，避免误判。
- 地址归一化要谨慎（同一小区不同楼栋不应简单合并）。
